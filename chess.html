  • // Description
  • Chess was a project I used to learn Unity, a popular game engine.
  • This game only allows users to move pieces in legal positions, and will track the location of all pieces on the board.
  • This game is not available for public access.


  • // Features
  • This game is not aimed towards being a fully functional chess simulator, buta way for me to learn Unity and the language it uses, C#.

    • Movable Pieces
    • Sound Effects
    • Highlighted Movable Tiles


  • // Development
  • I will be very straight-forward about this at the beginning---this project does not test for checks, checkmates, or stalemates. This project was a way for me to learn how Unity worked, and as such I focused more on the aspect of having interactable pieces and other game aspects, such as sound effects.
  • I wanted to learn how Unity worked because I want to work in game development. I must admit, it was quite a challenge. I had prior Java knowledge, but no concept whatsoever as to how a game engine worked. To accomplish my task, I had to look up countless resources regarding how C# interfaced with Unity, and how the game engine analyzed the scripts. This meant I had to self-learn a new library, but this wasn't an issue for me.
  • This was by far the most challenging project I ever attempted. However, I am proud of the fact that I was able to make an interactable game, and one that sported some unique features from other chess emulators, like Chess.com.


  • // Usage
  • By launching chess.exe, the game is playable with white moving first.
